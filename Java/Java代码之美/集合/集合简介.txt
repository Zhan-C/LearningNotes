集合分为两个父接口集合
	
	Collection:特点无序不唯一

	Map:分为键和值，以Set为键，Conllection为值，他的键为无序唯一，值为无序不唯一


子类接口

	Conllection：
			Set:特点无序，而且唯一
				
				HashSet：
						实现：使用哈希算法来根据哈希插入，值唯一，不能出现相同的数据，
							  否则会被覆盖，除非在类中进行去重
						特点：增删改查都比较快，但是占用内存比较高，稍微耗费内存
				TreeSet：
						实现：他是会使用根据插入的值进行运算，红黑树机制，会将数据的值进行比较
						特点：增加较慢，查询速度快
				
			List:有序，不唯一

				ArrayList:
						实现：底层使用的数组进行构建
					  	
						特点：他的查询速度比较快，但是插入比较慢，因为是使用数组进行构建
							  所以发生改变的时候会对数组进行修改，会创建内存所以增加删除较慢

				LinkdList:
						实现：底层使用链表进行构建
						
						特点：他是使用链表进行构建的，像链接起来的一张张表，如果插入就从中间插入一个
							  他会把左右的数据链接起来所以增加快，而删除的话他会把中间的链去掉把左右的数据链接起来，
							  所以删除也比较快，但是查询比较慢，因为他是基于链表，一张一张查询的，所以查询效率不高
